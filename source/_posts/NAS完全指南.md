## 什么是 NAS

**NAS（Network Attached Storage，网络附加存储）** 是一种通过局域网（LAN）或互联网提供文件存储与共享服务的专用设备。

简单来说，NAS 就是一台**连接在网络上、可以被多人同时访问的私人硬盘**。

```
         局域网 / 互联网
              │
    ┌─────────┼──────────┐
    │         │          │
  电脑      手机       平板
    │         │          │
    └────── NAS ─────────┘
         (文件存储中心)
```

### NAS 与普通移动硬盘的区别

| 对比项       | 移动硬盘        | NAS                  |
| ------------ | --------------- | -------------------- |
| 访问方式     | 物理连接（USB） | 网络访问（随时随地） |
| 同时访问人数 | 1人             | 多人并发             |
| 自动运行     | 需要手动插拔    | 24小时待机           |
| 数据冗余保护 | 无              | 支持 RAID            |
| 远程访问     | 不支持          | 支持                 |
| 扩展性       | 受限            | 可随时扩容           |

## NAS 的核心优势

### ✅ 数据自主可控

数据存储在自己家里，不依赖第三方云服务（如 iCloud、百度网盘），隐私更有保障，也不受服务商涨价或停服影响。

### ✅ 低成本大容量

以购买一块 4TB 企业级硬盘（约 ¥600）为例，1GB 成本约 ¥0.15，远低于各类云存储订阅费用。

### ✅ 局域网高速传输

通过千兆或万兆网络，局域网内传输速度可达 **100MB/s～1000MB/s**，远超网盘下载速度。

### ✅ 数据冗余保护

配合 RAID，即使一块硬盘损坏，数据也不会丢失，大幅降低数据损失风险。

### ✅ 功能极度可扩展

现代 NAS 系统（如群晖 DSM）拥有丰富的应用生态，可作为：

- 私有云 / 家庭媒体中心 / 下载机 / 开发服务器...

## NAS 的主要用途

### 1. 📁 家庭文件存储与共享

将所有家庭成员的文档、照片、视频统一存放在 NAS 上。每个成员拥有独立账号与权限，既共享又互不干扰。

**典型场景：**

- 家庭相册统一备份，手机拍的照片自动上传
- 家人共享电影资源库
- 工作文档随时从不同设备访问

### 2. 📷 照片/视频自动备份

取代 iCloud、Google Photos，打造**自托管照片管理系统**：

- **群晖 Moments / Photos**：媲美 Google Photos 的人脸识别、AI 相册管理
- **Immich**（开源）：高度还原 Google Photos 体验，可自行部署
- **PhotoPrism**：基于 AI 的照片整理

### 3. 🎬 家庭影音媒体服务器

配合媒体软件，将 NAS 变成家庭影院中心：

| 软件                   | 特点                              |
| ---------------------- | --------------------------------- |
| **Plex**               | 功能强大，支持转码，有手机/TV App |
| **Jellyfin**           | 完全开源免费，无需订阅            |
| **Emby**               | 界面美观，部分功能收费            |
| **群晖 Video Station** | DSM 内置，操作简单                |

### 4. ⬇️ 下载机（PT / BT / 离线下载）

让 NAS 24小时在后台下载，不占用电脑资源：

- **qBittorrent / Transmission**：BT/PT 下载
- **Aria2**：多协议下载（HTTP/BT/磁力）
- **群晖 Download Station**：图形界面友好

### 5. ☁️ 私有云同步（取代百度网盘/Dropbox）

- **Synology Drive**：完整复刻 Dropbox 体验，支持多设备同步
- **Nextcloud**：功能最全的开源私有云，支持文档协作
- **Seafile**：注重性能，适合大量小文件

### 6. 💻 代码与项目备份

开发者可以在 NAS 上部署 Git 服务：

- **Gitea**：轻量级 Git 平台，类似私有 GitHub
- **GitLab**（资源要求较高）

### 7. 🔄 系统/虚拟机备份

- **Active Backup for Business**（群晖）：备份 PC、Mac、服务器
- **Time Machine 目标**：为 Mac 提供 Time Machine 备份
- **Hyper Backup**：NAS 数据异地备份

### 8. 🖥️ 运行 Docker 容器 / 虚拟机

高性能 NAS 可运行各类服务：

```
常见 Docker 应用：
- Home Assistant（智能家居）
- Bitwarden（密码管理器）
- Pi-hole（广告拦截）
- Nginx Proxy Manager（反向代理）
- Portainer（Docker 管理界面）
```

## 主流 NAS 品牌与选购建议

### 品牌对比

| 品牌                   | 代表型号         | 系统        | 特点                   | 适合人群      |
| ---------------------- | ---------------- | ----------- | ---------------------- | ------------- |
| **群晖 Synology**      | DS223, DS923+    | DSM         | 生态最完整，操作最友好 | 所有用户首选  |
| **威联通 QNAP**        | TS-233, TVS-h874 | QTS         | 硬件性能强，功能多     | 进阶用户      |
| **铁威马 TerraMaster** | F2-423, F4-423   | TOS         | 性价比高               | 预算有限用户  |
| **自组 NAS**           | -                | TrueNAS/OMV | 完全自定义             | 折腾党 / 企业 |

### 选购要点

**盘位数量：**

- **2盘位**：家庭入门，最多两块硬盘，支持 RAID 1
- **4盘位**：家庭进阶/SOHO，灵活配置 RAID 5/6
- **6盘位及以上**：重度用户 / 小型企业

**处理器：**

- 仅文件存储：入门 ARM 处理器即可
- 需要 4K 转码 / Docker / 虚拟机：建议 Intel Core 系列（J4125 起步，i3/i5 更佳）

**内存：**

- 基础使用：2-4GB
- 运行多个 Docker 容器：8GB+
- 虚拟机：16GB+

**网络接口：**

- 千兆网口：日常够用
- 2.5G/10G 网口：大文件传输、视频剪辑素材访问

## 硬件组成详解

```
NAS 设备
├── 机箱（散热设计很重要）
├── 主板（含 CPU、内存）
├── 硬盘槽（2/4/6/8盘位）
│   ├── 系统盘（部分机型内置 DOM/eMMC）
│   └── 存储盘（用户自行购买）
├── 网络接口（1G/2.5G/10G）
└── USB 接口（外接硬盘/UPS）
```

### 硬盘选择建议

推荐选用专为 NAS 设计的**企业级/NAS 专用盘**：

| 品牌     | 系列                    | 特点                   |
| -------- | ----------------------- | ---------------------- |
| 西部数据 | WD Red Plus / Red Pro   | 性价比高，7x24小时设计 |
| 希捷     | IronWolf / IronWolf Pro | 内置健康监测，故障率低 |
| 东芝     | N300                    | 性价比选择             |

> ⚠️ **避免使用**桌面级硬盘（WD Blue/Seagate Barracuda）长期在 NAS 中运行，不适应 24小时振动与连续读写。

## RAID 模式详解

RAID（独立磁盘冗余阵列）通过多块硬盘的组合方式提供性能提升或数据冗余保护。

### 常用 RAID 级别

| RAID 类型           | 最少磁盘数 | 可用容量 | 可承受损坏数 | 特点                 |
| ------------------- | ---------- | -------- | ------------ | -------------------- |
| **JBOD**            | 1          | 100%     | 0            | 无冗余，容量叠加     |
| **RAID 0**          | 2          | 100%     | 0            | 速度快，无保护       |
| **RAID 1**          | 2          | 50%      | 1            | 完全镜像，最安全     |
| **RAID 5**          | 3          | (n-1)/n  | 1            | 均衡选择，最常用     |
| **RAID 6**          | 4          | (n-2)/n  | 2            | 双校验，更安全       |
| **SHR**（群晖专属） | 1          | 灵活     | 1-2          | 支持混搭不同容量硬盘 |

### 如何选择？

```
只有 2 块盘  →  RAID 1（镜像保护）
有 3 块盘    →  RAID 5（容量与保护均衡）
有 4+ 块盘   →  RAID 5 或 RAID 6（更强保护）
硬盘大小不一 →  群晖 SHR
```

> 🔴 **重要提示：RAID ≠ 备份！**  
> RAID 保护的是硬件故障，无法防范误删、勒索病毒、火灾等情况。  
> 请遵循 **3-2-1 备份原则**：3份数据 / 2种介质 / 1份异地。

## NAS 的初始配置

以**群晖 DSM** 为例（其他品牌流程类似）：

### 第一步：硬件安装

1. 将硬盘安装到硬盘托架
2. 连接网线（建议直连路由器）
3. 连接电源，开机

### 第二步：系统初始化

```
浏览器访问：http://find.synology.com
或直接访问：http://[NAS的IP地址]:5000
```

1. 安装 DSM 系统（约 10 分钟）
2. 创建管理员账号
3. 配置存储池和存储空间（选择 RAID 模式）
4. 完成基础向导

### 第三步：网络配置

```
控制面板 → 网络 → 网络界面
推荐设置：
- 为 NAS 分配固定 IP（静态 IP 或路由器绑定 MAC）
- 启用 IPv6（可选）
```

### 第四步：账户与权限

```
控制面板 → 用户账号与群组
- 为每位家庭成员创建独立账号
- 设置共享文件夹权限（读取/读写/无权限）
```

### 第五步：远程访问配置

**方法一：QuickConnect（最简单）**

```
控制面板 → QuickConnect
启用后获得专属 ID，如：https://QuickConnect.to/你的ID
```

**方法二：DDNS + 端口映射**

```
控制面板 → 外部访问 → DDNS
→ 注册 Synology DDNS 域名（*.synology.me）
→ 在路由器设置端口映射（5001 → NAS IP:5001）
```

**方法三：VPN（最安全）**

```
套件中心 → 安装 VPN Server
→ 配置 OpenVPN 或 WireGuard
→ 在外网通过 VPN 连接后访问内网 NAS
```

## 常见使用场景与操作方法

### 📂 Windows 映射网络驱动器

```
文件资源管理器 → 此电脑 → 映射网络驱动器
输入地址：\\192.168.1.100\shared_folder

或使用 SMB 协议：
控制面板 → 文件服务 → SMB → 启用
```

### 🍎 macOS 连接 NAS

```
Finder → 前往 → 连接服务器（⌘K）
输入：smb://192.168.1.100
或：afp://192.168.1.100（AFP 协议）
```

### 📱 手机访问（群晖 DS File App）

1. 下载 **DS File**（iOS/Android）
2. 输入服务器地址（局域网 IP 或 QuickConnect ID）
3. 输入账号密码即可浏览文件

### ⬆️ 照片自动同步（群晖 Moments）

1. NAS 安装 **Synology Photos**
2. 手机安装 **Synology Photos App**
3. 开启"自动备份"，手机拍照即时上传至 NAS

### 🎬 Plex 媒体服务器配置

```
套件中心 → 安装 Plex Media Server
→ 添加媒体库（电影/电视/音乐）
→ 指向 NAS 上的媒体文件夹
→ 手机/平板安装 Plex App 即可观看
```

### 🐳 Docker 部署示例（以 Nginx 为例）

```bash
# 群晖 Container Manager → 项目 → 新增
# 或 SSH 进入 NAS 执行：

docker run -d \
  --name nginx \
  -p 8080:80 \
  -v /volume1/docker/nginx:/usr/share/nginx/html \
  nginx:latest
```

## 安全配置建议

### 🔐 基础安全

- [ ] **修改默认 SSH 端口**（默认 22，改为 10022 等非标准端口）
- [ ] **禁用 admin 账号**，使用自定义管理员名称
- [ ] **启用两步验证（2FA）**：控制面板 → 用户 → 2 步骤验证
- [ ] **启用防火墙**：控制面板 → 安全性 → 防火墙
- [ ] **开启自动封锁**：多次登录失败自动封锁 IP

### 🔒 网络安全

- [ ] **关闭不必要的对外端口**，仅开放需要的服务
- [ ] **使用 HTTPS**（DSM 5001 端口），避免明文传输
- [ ] **申请免费 SSL 证书**：控制面板 → 安全性 → 证书 → Let's Encrypt
- [ ] **考虑使用 VPN**而非直接暴露 NAS 端口到公网

### 💾 数据安全

- [ ] **定期快照**（群晖 Snapshot Replication）
- [ ] **异地备份**：将重要数据备份到另一台 NAS 或云存储（Backblaze B2）
- [ ] **开启勒索病毒防护**（群晖 Storage Analyzer）

## 常见问题与排查

### Q：NAS 噪音很大怎么办？

- 更换低噪音风扇（如 Noctua 品牌）
- 在 DSM 中调低风扇转速（控制面板 → 硬件和电源 → 风扇速度）
- 将 NAS 放置在通风但隔音的储物间

### Q：硬盘读写指示灯一直在闪是什么问题？

- 正常现象：系统索引、健康扫描（后台任务）
- 检查方法：控制面板 → 任务计划 → 查看正在运行的任务

### Q：外网访问速度很慢？

- 检查家庭宽带的**上行带宽**（上传速度决定外网访问速度）
- 使用 DDNS + 端口映射比 QuickConnect 更快
- 考虑在路由器端启用 QoS 优先级

### Q：文件被误删如何恢复？

- 查看**回收站**（共享文件夹 → 启用回收站功能）
- 使用**快照**恢复（需提前配置 Snapshot Replication）
- 第三方数据恢复软件（如 R-Studio）

### Q：NAS 耗电量大吗？

- 典型 2盘位 NAS（如 DS223）：约 **15-20W**（工作状态）
- 休眠状态：约 5-8W
- 全年电费（按 ¥0.6/度）：约 **¥80-¥100**

## 进阶玩法

### 🔧 SSH 命令行访问

```bash
# 启用：控制面板 → 终端机和SNMP → 启动SSH
ssh admin@192.168.1.100 -p 22

# 常用命令
df -h              # 查看磁盘使用情况
top                # 查看系统资源
synoinfo get       # 查看系统信息
```

### 🐳 Docker Compose 部署 Nextcloud

```yaml
# /volume1/docker/nextcloud/docker-compose.yml
version: '3'
services:
  nextcloud:
    image: nextcloud:latest
    ports:
      - "8080:80"
    volumes:
      - ./data:/var/www/html
    environment:
      - MYSQL_HOST=db
      - MYSQL_DATABASE=nextcloud
      - MYSQL_USER=nextcloud
      - MYSQL_PASSWORD=your_password
  db:
    image: mariadb:latest
    environment:
      - MYSQL_ROOT_PASSWORD=root_password
      - MYSQL_DATABASE=nextcloud
      - MYSQL_USER=nextcloud
      - MYSQL_PASSWORD=your_password
    volumes:
      - ./db:/var/lib/mysql
```

### 📊 监控与告警

- **群晖 SNMP**：配合 Grafana + InfluxDB 实现可视化监控
- **DS Finder App**：手机接收 NAS 告警推送（硬盘温度过高、RAID 降级等）
- **邮件通知**：控制面板 → 通知 → 配置 SMTP 邮件告警

### 🌐 反向代理配置

通过一个域名访问 NAS 上的多个服务：

```
控制面板 → 登录门户 → 反向代理 → 新增

示例：
  来源：https://plex.yourdomain.com
  目标：http://localhost:32400

  来源：https://nextcloud.yourdomain.com  
  目标：http://localhost:8080
```
